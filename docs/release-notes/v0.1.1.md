# firefox-cli 0.1.1

First signed release of `firefox-cli`: a terminal CLI, native messaging host, and Firefox extension for controlling your normal Firefox session from command-line tools and AI agents.

## Highlights

- Control the active Firefox session without launching a separate automation profile.
- Install a signed self-hosted Firefox extension, `FF-CLI Bridge`, approved by Mozilla Add-ons for unlisted distribution.
- Pair the extension with the local native messaging host and verify setup with `firefox-cli doctor`.
- Inspect tabs, windows, page state, DOM snapshots, element refs, console/errors, network observations, cookies, storage, clipboard, downloads, and browser capabilities.
- Drive browser actions including navigation, tab/window selection, clicks, form filling, keyboard input, scrolling, drag/drop, file upload, waits, screenshots, and serial batch workflows.
- Receive structured JSON output and protocol errors for agent/tool integrations.

## Install

```sh
npm install -g firefox-cli
firefox-cli setup
firefox-cli setup native-host
firefox-cli doctor
```

`firefox-cli setup` prints the signed extension path and native-host setup guidance.

## Distribution

- Signed extension XPI: `firefox-cli-0.1.1.xpi`
- AMO review source archive: `firefox-cli-0.1.1-source.zip`
- Self-hosted extension update manifest: https://opensource.respawn.pro/firefox-cli/updates.json

The production extension ID is `ff-cli-bridge@respawn.pro`.

## Security

Approving the extension grants the paired local CLI full control over reachable Firefox tabs and windows, including authenticated sites. Only approve pairing on machines and user accounts you trust.

## Known Limits

- Private windows are readable for listings and diagnostics, but mutating commands are rejected.
- Full-page screenshots, PDF save automation, network route/mock/block, HAR export, Chrome CDP, and browser-internal privileged pages are unsupported.
- DOM snapshots and element refs target the main frame; iframe support is diagnostic/read-only.
