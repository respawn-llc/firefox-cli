## Audit Scope

This repository is small enough for a full architectural pass: roughly 35k lines of TypeScript across `packages` and `scripts`. Generated `dist` output was not treated as source. `bun audit` reports no known vulnerabilities. `bun outdated` shows non-blocking major-version drift for TypeScript, Vite, and Node type packages.

## Critical Workstreams

- [x] Harden native-host local IPC, persisted state, and transport lifecycle as one reliability slice. In `packages/native-host/src/local-ipc.ts`, `packages/native-host/src/local-ipc-frame.ts`, `packages/native-host/src/native-host-runtime.ts`, `packages/native-host/src/native-messaging-frame.ts`, `packages/native-host/src/pair-state.ts`, and `packages/native-host/src/native-manifest.ts`, build a shared native-host reliability layer instead of fixing socket cleanup, state writes, timeouts, and frame errors separately. Dependency order: create atomic file and lock utilities, add endpoint ownership checks and current-user isolation, add client/server deadlines, then isolate malformed frames per read loop. Cover Unix stale socket ownership, Windows named-pipe ACL behavior, concurrent host startup, non-socket files at the socket path, torn JSON/token writes, host response hangs, and malformed native-messaging frames.

## High Workstreams

- [x] Consolidate CLI command contracts into one typed command specification. In `packages/protocol/src/registry/*.ts`, `packages/cli/src/route-registry.ts`, `packages/cli/src/parse.ts`, `packages/cli/src/format.ts`, `packages/cli/src/runner.ts`, and `packages/cli/src/commands/*.ts`, replace duplicated route ids, help text, option classification, JSON-output detection, unsupported capability gates, formatter switches, and parser loops with one command-spec layer. Dependency order: define the spec shape and migration tests, generate strict route matching and unknown-option rejection from it, move request builders behind typed parser schemas, then key output formatters and capability gates from the same source. Include batch argv parsing in the migration so nested command behavior cannot drift.

- [x] Replace cast-heavy runtime dispatch with compile-time checked handler maps. `packages/cli/src/format.ts`, `packages/extension/src/browser-dispatch.ts`, `packages/extension/src/browser-handlers/*.ts`, `packages/extension/src/content-snapshot/router.ts`, `packages/native-host/src/host-broker.ts`, and `packages/protocol/src/envelopes.ts` repeatedly narrow commands through string checks followed by `as RequestEnvelope<...>` or `as ResponseEnvelope<...>`. Add typed dispatcher helpers that narrow by command id, require handler coverage for each protocol-owned command, and fail CI when a command is registered without a CLI parser, browser handler, content handler, formatter, or explicit unsupported marker.

- [x] Split extension content control into lifecycle-safe services. `packages/extension/src/content-actions.ts`, `packages/extension/src/content-dom-adapter.ts`, `packages/extension/src/content-snapshot/dom.ts`, `packages/extension/src/content-wait.ts`, `packages/extension/src/content-snapshot/log-capture.ts`, `packages/extension/src/content-snapshot.ts`, and `packages/extension/src/background.ts` combine command dispatch, element resolution, DOM mutation, event synthesis, fabricated browser objects, import-time log capture, and bootstrap side effects. Extract a shared element resolver, split action families from event/file/data-transfer primitives, move jsdom-only fallbacks behind explicit compatibility adapters, and introduce `startBackground`/`startContentScript` lifecycle factories with teardown handles. Treat console/error capture as a service with a documented content-script versus page-world contract.

- [x] Bound long-lived extension state and privilege-sensitive command execution. In `packages/extension/src/network-tracker.ts`, `packages/extension/src/background.ts`, `packages/extension/src/eval-executor.ts`, `packages/extension/src/content-wait.ts`, and `packages/protocol/src/registry/*.ts`, add lifecycle pruning for network maps and explicit metadata/tests for dangerous commands. Wire tab removal and stale active-request sweeps into the network tracker. Add command metadata for broad-permission operations such as `<all_urls>`, clipboard, downloads, cookies, `eval`, and `wait --fn`; assert that unapproved, malformed, or wrong-boundary requests cannot reach those handlers. Preserve the full-control post-approval MVP behavior.

- [x] Decompose oversized production files along package ownership boundaries. Split `packages/extension/src/background-controller.ts`, `packages/native-host/src/local-ipc.ts`, `packages/protocol/src/envelopes.ts`, and `packages/protocol/src/browser.ts` into transport/session/state/schema modules that map directly to CLI, native-host, extension, and protocol responsibilities. Do this after the native-host reliability and command-contract workstreams have started so extracted modules land around the new abstractions instead of preserving the current seams.

## Decision Tickets

- [x] Decide the protocol compatibility and content-script refresh policy before refactoring negotiation. `packages/protocol/src/envelopes.ts` hardcodes feature-version checks, while `packages/native-host/src/native-host-runtime.ts`, `packages/extension/src/background-controller.ts`, `packages/native-host/src/local-ipc.ts`, and `packages/native-host/src/host-broker.ts` duplicate session state machines. Decide whether required protocol versions live entirely in command metadata, how negotiated CLI-host and host-extension sessions compose, and whether background should reinject content scripts on `VERSION_MISMATCH`. Implement the refactor only after this policy is documented in the plan.

- [x] Decide iframe command scope as a product contract. Docs and architecture mention frame-scoped work, but implementation lists iframe diagnostics and injects content scripts with `allFrames: false` in `packages/extension/src/background.ts`. Decide whether iframe actions are unsupported, prototype-gated, or first-class frame targets. Encode that decision in protocol capability metadata, CLI help, snapshot diagnostics, and tests.

## Medium Workstreams

- [x] Make script and release automation safer and more schema-driven. In `scripts/package.ts`, `scripts/package-check.ts`, `scripts/release-check.ts`, `scripts/sign-extension.ts`, `scripts/e2e-phase2.ts`, `scripts/e2e-disposable-firefox.ts`, `scripts/e2e-disposable-workflow.ts`, and `scripts/manifest-validation.ts`, replace unguarded generated-directory cleanup, ad-hoc JSON parsing, and structure-only XPI trust checks with shared utilities. Add a safe generated-artifact reset helper that verifies targets under repository `dist`, a `runCliJson`/`parseJsonWithSchema` utility with zod schemas and stdout context, and release checks that prove signed artifact provenance or require an explicit non-release local override.

- [x] Make script process management platform-aware and tree-safe. `scripts/process-runner.ts`, `scripts/e2e-disposable-firefox.ts`, `scripts/e2e-firefox-cleanup.ts`, and `scripts/marionette-client.ts` use POSIX-style signals, broad `ps` parsing, and command-substring matching. Add platform process adapters, use process groups on POSIX and `taskkill /T` or equivalent on Windows, and verify descendant process termination before E2E scripts exit.

- [ ] Replace allocation-heavy frame parsers with cursor-based buffering. `packages/native-host/src/native-messaging-frame.ts` repeatedly concatenates buffers while filling frames, and `packages/native-host/src/local-ipc-frame.ts` buffers line chunks similarly. Use a retained buffer with read offsets or a small frame-reader abstraction that enforces byte caps before copying, copies only completed frames, and has stress tests for adversarial chunking near size limits.

## Lower Workstreams

- [ ] Split oversized test suites and add focused module coverage. `packages/cli/src/cli.test.ts`, `packages/protocol/src/protocol.test.ts`, `packages/extension/src/content-snapshot.test.ts`, `packages/extension/src/browser-commands.test.ts`, `packages/extension/src/content-actions.test.ts`, and `scripts/package-check.test.ts` are large enough to hide coupling. Add focused `parse.test.ts`, `format.test.ts`, command-builder tests, content-action adapter tests, and protocol registry tests, then keep one small boundary suite per runtime crossing.

- [ ] Consolidate duplicated timeout, sleep, and polling helpers in scripts. `scripts/marionette-client.ts`, `scripts/e2e-disposable-workflow.ts`, `scripts/e2e-disposable-firefox.ts`, `scripts/e2e-phase2.ts`, and `scripts/process-runner.ts` each carry similar timeout or sleep utilities. Move them into a shared script utility with cancellation, consistent timeout messages, and output diagnostics.

- [ ] Add a dependency upgrade lane with explicit release-age rules. `bun audit` is clean, but installed versions trail latest major releases for TypeScript, Vite, and Node type definitions. Add a scheduled check that records allowed minimum release age, differentiates safe patch/minor upgrades from major migrations, and runs `bun run check` plus `bun run release:check` after upgrades.
